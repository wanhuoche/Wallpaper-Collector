### 2. JWT Secret 强制环境变量 ✅
- **问题**：`server/middleware/auth.js`、`functions/api/[[route]].js`、`worker/src/jwt.js` 都有硬编码的 fallback secret，忘记设环境变量时所有环境共用同一个 JWT 签名密钥。
- **方案**：三处均移除硬编码 fallback，改为启动/首次请求时检查 `JWT_SECRET`，不存在则抛出明确错误。
- **状态**：已完成
  - `server/middleware/auth.js` — 模块加载时检查 `process.env.JWT_SECRET`，无则 `throw`
  - `functions/api/[[route]].js` — 懒初始化 + `getSecret()` / `setSecret(env)`，首次请求初始化
  - `worker/src/jwt.js` + `index.js` — 同上模式，`fetch` 入口调用 `setSecret(env)`

### 3. 收藏同步改为"先拉后合"，防止云端数据被空本地覆盖 ✅
- **问题**：`favorites.js:229` — 页面初始化时先加载 `localStorage` 的本地收藏列表，再和云端同步。如果用户换设备/浏览器，本地收藏为空，虽然服务端合并逻辑正确，但 init 流程缺少显式的"先拉"步骤。
- **方案**：
  - `init()` 时先调用 `GET /api/auth/favorites` 拉取云端列表作为基准。
  - 再将本地列表与云端做双向合并（按 URL 去重，按 `savedAt` 取最新）。
  - 合并结果同时写回本地（localStorage）和云端（POST sync）。
- **状态**：已完成
  - 新增 `pullFromCloud()` — init 时先从云端拉取
  - 新增 `mergeLocal(cloudFavs)` — 客户端合并，URL 为键、savedAt 比新
  - `app.js` init 改为 `await pullFromCloud()` → `syncWithCloud()`
- **参考文件**：`js/favorites.js`、`js/app.js`

### 4. 移除生产环境调试代码 ✅

## 🟢 低优先级

### 12. 统一后端实现（精简为单套）
- **问题**：`server/`（Node.js Express）、`worker/`（Cloudflare Worker）、`functions/`（Pages Functions）三套后端并行维护，功能相同但依赖不同，改动一处需要同步三处。
- **方案**：选定一套作为主力（推荐 `server/` 最简单或 `functions/` 最轻量），另外两套归档或删除。

### 13. IIFE 模块迁移到 ES Modules
- **问题**：模块间通过 `window.WallpaperApp` 全局命名空间通信，加载顺序完全依赖 HTML 中 `<script>` 标签顺序。顺序错了就是静默运行时错误，排查困难。
- **方案**：改为 `<script type="module">` + `import/export`，依赖关系由引擎解析，不再依赖标签顺序。可渐进迁移，先改一个文件验证。

### 14. 虚拟滚动 / DOM 节点复用
- **问题**：搜索结果全部通过 `innerHTML` 渲染到 DOM，500 张图片全部挂载时内存占用高、首屏性能下降。`loading="lazy"` 只能延迟加载图片资源，DOM 节点本身仍在文档中。
- **方案**：
  - 方案 A：简单分页，DOM 中仅保留当前页 ± 1 页的节点，其余移除。
  - 方案 B：虚拟滚动（如只渲染可视区 ± 2 行的卡片），但壁纸网格高度不一，实现复杂度高。

### 15. 翻译缓存
- **问题**：Wallhaven 源的中文关键词每次都调 MyMemory 翻译 API，重复搜索浪费请求且依赖外部服务可用性。
- **方案**：`Map` 或 `localStorage` 缓存翻译结果（key: 中文词 → value: 英文词），命中缓存直接跳过 API 调用。缓存上限设 200 条，超出删最旧的。

### 16. 批量下载 / 多选
- **问题**：一张张点下载效率低。
- **方案**：多选模式（勾选卡片），一键打包下载或复制链接列表。打包用 JSZip，跨域限制需 Worker 代理配合。

### 17. 导出收藏夹
- **问题**：收藏无法导出分享或备份。
- **方案**：导出为 HTML 画廊页面或 JSON，方便分享和备份。

### 18. 图片加载优化
- **问题**：滚动多了已出视口的图片仍占内存。
- **方案**：`loading="lazy"` 已有，可配合虚拟列表进一步优化。但壁纸网格卡片高度不一，虚拟列表可能引入跳动。

### 19. 回到顶部按钮
- **问题**：无限滚动加载多页后，回到顶部需要手动滚很久。
- **方案**：右下角浮动"回到顶部"按钮，滚动超过 2 屏时显示，点击平滑滚动到顶部。

### 20. 移动端适配优化
- **问题**：当前网格在大屏手机上固定 2 列，浪费空间；部分 UI 元素偏小不利于触控。
- **方案**：
  - 网格列数根据屏幕宽度自适应：< 480px 1 列、480-768px 2 列、768-1024px 3 列、> 1024px 4 列。
  - 增大触控目标至最小 44x44px。

### 21. 添加自动化测试
- **问题**：整个项目零测试覆盖，改动容易引入回归 bug（如之前的分类筛选大小写 bug）。
- **方案**：至少为核心逻辑添加单元测试：
  - `filterByCategory()` — 分类过滤
  - `parseResponse()` 各图源解析
  - 收藏合并算法
  - 筛选条件（比例/分辨率/内容分级）
- 测试框架可用 Vitest 或直接用 Node 内置 `node:test`。
