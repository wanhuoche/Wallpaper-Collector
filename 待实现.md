## 🔴 高优先级

### 1. JS 文件拆分 ✅
- **问题**：`script.js` 608 行塞了 13 个职责（配置、状态、搜索、渲染、设置、预览、下载…），后续加功能会更难维护
- **方案**：拆为 `js/source-config.js`（图源配置）、`js/search.js`（搜索/筛选/渲染/预览/下载）、`js/app.js`（状态/设置/快捷键/初始化）
- **状态**：已完成
- **实现细节**
  - 采用 IIFE 模块模式（`(function() { ... })()`），不引入构建工具，保持零依赖纯静态
  - 通过 `window.WallpaperApp` 全局命名空间在模块间共享 state、dom 引用、SOURCE_CONFIG、storage API
  - 加载顺序：`source-config.js` → `search.js` → `storage.js` → `app.js`（后面的依赖前面的挂载到 `W` 上的属性）
  - 同时新增了 `js/storage.js`，单独承担 IndexedDB / FSA / 导入导出的持久化逻辑
  - `script.js` 已删除

### 2. API Key 持久化 ✅
- **问题**：`localStorage` 保存设置在重启电脑后可能丢失
- **方案**：三层存储 — IndexedDB（日常）+ File System Access API（自动磁盘同步）+ 手动导入/导出 JSON（全浏览器兜底）
- **状态**：已完成
- **实现细节**
  - **IndexedDB**（`js/storage.js`）：数据库名 `wallpaper-db`，单 object store `settings`，存放设置 JSON + 目录句柄（dirHandle），浏览器清理数据时比 localStorage 更不易被清除
  - **File System Access API**：调用 `window.showDirectoryPicker()` 让用户选择一个本地目录，保存 `FileSystemDirectoryHandle` 到 IndexedDB；每次保存设置时自动写入 `壁纸收集器设置.json` 到该目录；重启时优先从此文件读取。仅 Chrome / Edge 支持
  - **手动导入/导出**：创建 `<a download>` / `<input type=file>` 触发下载/选择文件，全浏览器通用兜底，`localStorage` 不再写入，仅读取并迁移旧数据后立即清除

### 3. 搜索防抖 ✅
- **问题**：快速切换筛选标签会连续发起多个请求
- **方案**：筛选标签变更加入 300ms debounce + AbortController 取消上一个未完成请求
- **状态**：已完成
- **实现细节**
  - 在 `app.js` 中，三个筛选标签（比例/清晰度/内容分级）的 click 事件都调用 `filterSearch()`，该函数内部用 `setTimeout` 实现 300ms debounce
  - 在 `search.js` 的 `doSearch()` 中，每次发起新请求前通过 `AbortController.abort()` 取消上一个未完成的 fetch，fetch 的 `signal` 参数传入 `abortController.signal`
  - 被取消的请求在 catch 中通过 `err.name === 'AbortError'` 识别并静默退出
  - 翻译进行期间也能被取消（`signal.aborted` 检查）

### 4. 错误处理细化 ✅
- **问题**：网络断开、API 限流、密钥过期等场景统一提示，用户无法判断原因
- **方案**：根据 HTTP 状态码（429/401/5xx）给出不同提示
- **状态**：已完成
- **实现细节**
  - 在 `search.js` 的 `doSearch()` 中，`fetch` 返回 `!resp.ok` 时根据 `resp.status` 分类：
    - `401` → "API Key 无效或已过期，请在设置中更新"
    - `403` → "API 拒绝访问（403），请检查 API Key 权限"
    - `429` → "请求过于频繁，请稍后重试"
    - `5xx` → "图源服务器故障（{status}），请稍后重试"
    - 其他状态码 → 仍尝试从响应体 JSON 提取错误消息，无则 fallback 到通用 "请求失败 (xxx)"
  - `fetch` 完全失败（网络断开 / DNS 失败 / CORS 等）时，浏览器抛出 `TypeError: Failed to fetch`，在 catch 中检测 `err.message === 'Failed to fetch'` 并替换为 "网络连接失败，请检查网络后重试"

### 19. 登录/注册系统 ✅
- **问题**：无用户体系，收藏和设置无法跨设备同步
- **方案**：Node.js + Express + SQLite + JWT + bcrypt 后端，前端新增 login.html / register.html
- **状态**：已完成
- **实现细节**
  - **后端**：`server/` 目录，Express + better-sqlite3（零配置单文件数据库）+ jsonwebtoken + bcrypt
  - **数据库**：单表 `users`（id, username, email, password(bcrypt hash), avatar, created_at, updated_at），`server/db.js` 自动初始化
  - **API**：
    - `POST /api/auth/register` — 注册（用户名+邮箱+密码），返回 JWT + 用户信息
    - `POST /api/auth/login` — 登录（邮箱+密码），返回 JWT + 用户信息
    - `GET /api/auth/me` — 获取当前登录用户（需 Bearer Token）
    - `PUT /api/auth/password` — 修改密码（需旧密码验证）
    - `GET /api/health` — 健康检查
  - **安全**：密码 bcrypt cost=12（异步，避免阻塞事件循环）；JWT 7 天过期；输入校验（用户名 3-20 字符/邮箱格式/密码 8-16 位）；登录接口 IP 限流（每分钟 10 次）；错误信息不泄露具体原因
  - **前端**：
    - `login.html`：Apple 风格居中毛玻璃卡片 + Unsplash 背景大图，邮箱/密码表单，已登录自动跳转
    - `register.html`：同上风格，多一个用户名字段
    - `js/auth.js`：IIFE 模块挂载到 `W.auth`，包含 token 管理（localStorage）、API 封装（5s 超时）、登录态检查、导航栏用户 UI（头像圆圈 + 下拉菜单 + 退出按钮）
  - **页面流**：`index.html` 启动 → `checkAuth()` 验证 token → 无 token 跳转 `login.html` → 登录/注册成功存 token 跳回 `index.html` → 导航栏显示用户头像
  - **启动**：
    ```
    # 终端 1：后端
    cd server && npm start    # localhost:3000

    # 终端 2：前端（VS Code Live Server 或 npx serve）
    ```
  - **容错**：后端未启动时，API 请求 5s 超时 → 清除过期 token → 显示"登录"按钮，不影响壁纸搜索等核心功能
  - **加载顺序**：`source-config.js` → `search.js` → `storage.js` → `auth.js` → `app.js` → `favorites.js`
